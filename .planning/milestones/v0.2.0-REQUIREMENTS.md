# Requirements Archive: DMJedi v0.2.0

**Archived:** 2026-04-23
**Milestone:** v0.2.0
**Outcome:** shipped

## Archived Requirements

### Generator Infrastructure

- [x] **GEN-01**: Generator registry supports parameterized generators via factory pattern (dialect, mode)
- [x] **GEN-02**: SQL templates route all types through dialect-aware helpers (no hardcoded system column types)

### SQL Dialects

- [x] **DIAL-01**: User can generate DuckDB-compatible SQL from DVML models
- [x] **DIAL-02**: User can generate Databricks SQL-compatible output from DVML models
- [x] **DIAL-03**: User can generate PostgreSQL-compatible SQL from DVML models
- [x] **DIAL-04**: Generated SQL includes hash key columns with dialect-appropriate hash functions
- [x] **DIAL-05**: Generated SQL uses correct identifier quoting per dialect

### Streaming

- [x] **STRM-01**: User can generate streaming pipeline output from Spark Declarative generator
- [x] **STRM-02**: Streaming mode produces DLT streaming table definitions distinct from batch

### LSP Server

- [x] **LSP-01**: Editor shows parse errors and lint warnings as diagnostics in real-time
- [x] **LSP-02**: Editor provides keyword completions while typing DVML
- [x] **LSP-03**: Editor provides entity reference completions (hub names, satellite names, etc.)
- [x] **LSP-04**: Editor shows entity details on hover (attributes, relationships, source location)
- [x] **LSP-05**: Editor supports go-to-definition for entity references
- [x] **LSP-06**: Editor shows document symbol outline for `.dv` files

### LLM Integration

- [x] **LLM-01**: MCP server exposes validate tool for AI workflows
- [x] **LLM-02**: MCP server exposes generate tool for AI workflows
- [x] **LLM-03**: MCP server exposes model explanation tool for AI workflows
- [x] **LLM-04**: CLI supports `--format json` flag for structured output consumable by LLMs and CI/CD

### Testing

- [x] **TEST-01**: Integration tests execute generated DuckDB SQL against real in-memory DuckDB database
- [x] **TEST-02**: Canonical fixture data exists for all DV2.1 entity types (hubs, sats, links, nhsats, nhlinks, effsats, samlinks, bridges, PITs)
- [x] **TEST-03**: SQLGlot validates syntax of generated Databricks SQL output
- [x] **TEST-04**: Test suite enforces minimum coverage threshold

### Release

- [x] **REL-01**: Tagged GitHub release with changelog and install instructions
- [x] **REL-02**: Example DVML models and generated output for all supported targets (DuckDB, Databricks SQL, PostgreSQL, Spark Declarative)

## Requirement Outcomes

All v0.2.0 milestone requirements shipped. No milestone requirements were dropped.
