# Requirements: DMJEDI

**Defined:** 2026-04-08
**Core Value:** Users write `.dv` files and get validated, generated pipeline code for the full DV 2.1 entity specification.

## v0.1.1 Requirements

Requirements for milestone v0.1.1: Complete DV 2.1 Entity Coverage.

### Parser Improvements

- [ ] **PARSE-01**: Parser caches Lark instance as singleton instead of creating per call
- [ ] **PARSE-02**: Parse errors include source file, line, column, and contextual hint
- [ ] **PARSE-03**: Parser handles all new entity keywords without grammar ambiguity

### Data Types

- [ ] **TYPE-01**: User can use `bigint`, `float`, `varchar`, `binary` data types in field declarations
- [ ] **TYPE-02**: SQL Jinja generator maps new data types correctly per dialect (default, postgres, spark)
- [ ] **TYPE-03**: Spark Declarative generator maps new data types to PySpark types

### Raw Vault Entities

- [ ] **ENTITY-01**: User can declare `effsat` with `of` referencing a link, with user-declared temporal fields
- [ ] **ENTITY-02**: User can declare `samlink` with `master`/`duplicate` keywords referencing the same hub
- [ ] **ENTITY-03**: User can declare `nhsat` with `of` referencing a hub or link (current-state-only satellite)
- [ ] **ENTITY-04**: User can declare `nhlink` with `references` to 2+ hubs (current-state-only link)

### Query-Assist Constructs

- [ ] **QUERY-01**: User can declare `bridge` with `path` using arrow-chain syntax across hubs and links
- [ ] **QUERY-02**: User can declare `pit` with `of` hub anchor and `tracks` satellite list

### Linting

- [ ] **LINT-01**: Linter warns if `effsat` parent is not a link
- [ ] **LINT-02**: Linter warns if `samlink` master/duplicate don't reference the same hub
- [ ] **LINT-03**: Linter warns if entity names don't follow configurable naming convention (prefix/suffix)
- [ ] **LINT-04**: Resolver validates bridge path forms a valid chain through existing entities
- [ ] **LINT-05**: Resolver validates PIT satellites belong to the PIT's anchor hub

### Code Generation

- [ ] **GEN-01**: SQL Jinja generates correct SQL for all 6 new entity types
- [ ] **GEN-02**: Spark Declarative generates correct DLT Python for all 6 new entity types
- [ ] **GEN-03**: Non-historized entities generate MERGE/overwrite patterns (not INSERT)
- [ ] **GEN-04**: Bridge and PIT generate as views/SELECT (not CREATE TABLE)

### CLI

- [ ] **CLI-01**: User can pass `--dialect` flag to `dmjedi generate` for SQL Jinja dialect selection

### Documentation

- [ ] **DOC-01**: Markdown docs generator covers all 6 new entity types
- [ ] **DOC-02**: Docs generator produces Mermaid entity-relationship diagrams
