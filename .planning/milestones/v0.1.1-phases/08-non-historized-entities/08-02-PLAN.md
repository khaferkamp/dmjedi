USING src_{{ nhsat.name }} AS source
ON target.{{ nhsat.parent_ref }}_hk = source.{{ nhsat.parent_ref }}_hk
WHEN MATCHED THEN
    UPDATE SET
        target.load_ts = source.load_ts,
        target.record_source = source.record_source{% if nhsat.columns %},{% endif %}
{%- for col in nhsat.columns %}
        target.{{ col.name }} = source.{{ col.name }}{% if not loop.last %},{% endif %}
{%- endfor %}
WHEN NOT MATCHED THEN
    INSERT ({{ nhsat.parent_ref }}_hk, load_ts, record_source{% if nhsat.columns %}, {% endif %}{% for col in nhsat.columns %}{{ col.name }}{% if not loop.last %}, {% endif %}{% endfor %})

**Design note on column omission:** Unlike SQL Jinja templates (which explicitly list columns in MERGE SET/INSERT clauses), the Spark DLT generator relies on `dlt.apply_changes()` to infer the full schema from the source dataset at runtime. User-defined columns on NhSat/NhLink are validated at the model layer (resolver checks types exist, linter checks naming conventions) but are intentionally NOT enumerated in the generated DLT Python code. This matches the existing `_generate_satellite` pattern where `apply_changes()` also does not list individual satellite columns.
