<interfaces>
The current LSP already routes all diagnostics through `analyze_document()` and then publishes the returned `analysis.diagnostics` from `src/dmjedi/lsp/server.py`.

Gap-closing additions should stay inside that existing contract rather than inventing a second diagnostic path:

```python
@dataclass(slots=True)
class DocumentAnalysis:
    diagnostics: list[types.Diagnostic]
    declarations: dict[str, DeclarationInfo]

def analyze_document(uri: str, source: str, version: int | None) -> DocumentAnalysis: ...
```

Expected same-document semantic helper shape for this plan:

```python
def semantic_diagnostics(
    module: DVMLModule,
    declarations: dict[str, DeclarationInfo],
    source: str,
) -> list[types.Diagnostic]: ...
```

The implementation must stay current-document-only per D-01 through D-09. Do not introduce workspace caches, import resolution passes, or cross-file indexing.
</interfaces>
